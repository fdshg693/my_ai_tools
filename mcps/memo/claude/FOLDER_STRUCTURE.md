# Folder Structure

## Module structure

Files are split by domain (memo / user / category) and by layer: `infra/` (shared gateways — SQLite, OpenAI), `migrations/` (version-managed schema migrations), `repository/` (pure data access), `service/` (domain rules + orchestration), and `server/` (the transport edges — `server/mcp/` for the MCP server incl. `auth`/`authz`/`tools`, `server/web/` for the local admin web UI).

| Module | Responsibility |
|--------|---------------|
| `infra/database.py` | Shared infrastructure only. SQLite connection factory (`_connect_db`, **`PRAGMA foreign_keys=ON` on every connection**), `init_db()` (autocommit conn: `PRAGMA journal_mode=WAL` + `foreign_keys=ON` → `_create_schema()` → `run_migrations()` → `_seed()`). `_create_schema()` creates the current **FK** schema (`users` / `categories` / `memos` / `memo_embeddings`) via `CREATE TABLE IF NOT EXISTS` + indexes + `_create_triggers()`. `_create_triggers()` defines the two category↔memo sync triggers (`trg_categories_rename_cascade`, `trg_categories_delete_reassign`) — imported by `migrations.m001`. `_seed()` does the `admin` seed + per-user `OTHERS` seed + `(user, category)` back-fill. Constants `ADMIN_USER = "admin"` / `OTHERS_CATEGORY = "OTHERS"`. `DB_PATH` defaults to `parent.parent / "memo.db"` (= `src/memo/memo.db`), overridable via `MEMO_DB_PATH`. Holds no domain CRUD. |
| `migrations/` | Version-managed schema migrations in their own package (see [SCHEMA.md](./SCHEMA.md) "Changing the schema"). `runner.py`: `schema_version` table + `_MIGRATIONS` registry + `run_migrations(db)` (applies unapplied versions; needs an autocommit conn). `m001_foreign_keys.py`: converts a legacy no-FK DB to the FK schema (orphan cleanup + table rebuild + triggers + `foreign_key_check`), idempotent. `__main__.py`: standalone `memo-migrate` / `python -m memo.migrations [DB_PATH]` entry (calls `init_db()`, prints `schema_version` before→after). `__init__.py` re-exports `run_migrations`. `infra/database.init_db()` imports `run_migrations` lazily to avoid the `database ↔ migrations` import cycle. |
| `repository/memo.py` | Pure data access for memos (`*_memo_db`). **Always scoped by `user`** (no `is_admin` / no cross-user mode): get/update/delete use `_scope(memo_id, user)`; list/count/search use `_base_filter(user, category)`. `list_memos_db` also takes `offset` for paging; `count_memos_db(user, category)` returns the total for the same filter (web memo-list pager). The category canonicalization rule lives in `repository/category.py`; `create_memo_db`/`update_memo_db` import `normalize_category()` from there and apply it on every write (but do **not** validate category existence — that is `service.memo`'s job, so this layer stays permissive). Uses only `_connect_db` (+ `repository.category` for normalization); no authorization. |
| `repository/category.py` | Pure data access for the first-class per-user category ledger (`categories`). `normalize_category()` (trim + uppercase, empty/`None` → `OTHERS = OTHERS_CATEGORY`) — the canonical-form rule imported by `repository.memo` (one-way `memo → category`). CRUD helpers: `list_categories_db(user)` (returns dict rows, name-sorted), `get_category_db`, `category_exists_db` (memo-validation check), `create_category_db` (UNIQUE → None on dup), `ensure_default_category_db` (seed `OTHERS`), `rename_category_db` / `delete_category_db` — these two touch **only the `categories` row**; the cascade to `memos` (rename repoints `memos.category`; delete reassigns linked memos to `OTHERS`) is done **by the DB triggers** (`trg_categories_*` in `infra.database._create_triggers`), not raw SQL here. Imports only from `infra` (`OTHERS_CATEGORY`, `_connect_db`); no `repository.memo` import (no `memo ↔ category` cycle). No authorization (categories are per-user; `admin` is not special). |
| `repository/user.py` | Data access for the user ledger (`users`, `*_user_db`) plus the registration check (`is_registered_user`). `name` is the immutable identifier; `display_name`/`note` are editable. `delete_user_db` deletes **only the `users` row**; the user's `categories`, `memos`, and (transitively) `memo_embeddings` are removed by the **FK `ON DELETE CASCADE`** (no hand-written cross-table SQL; no orphan data left). |
| `repository/embedding.py` | Pure data access for the embedding cache (`memo_embeddings`): `get_cached_embedding` / `upsert_embedding` (UPSERT) / `delete_embedding`. vector is a JSON array. `upsert_embedding` now requires the target memo to exist (FK `memo_id → memos(id)`); rows are auto-dropped when the memo is deleted (cascade). No network, no authorization. |
| `infra/embedding.py` | OpenAI embedding API wrapper (leaf). The only place that imports `openai` and reads `OPENAI_API_KEY`. Provides `embed_text(text)`, constant `MODEL` (overridable via `MEMO_EMBEDDING_MODEL`), and exception `EmbeddingError`. The key is fetched lazily at call time. On import it loads `mcps/memo/.env` (fixed `parents[3]/.env`) via `python-dotenv` (existing env vars take precedence; .env does not override). |
| `service/memo.py` | Memo domain layer that **both transport edges go through** (MCP tools + web UI never call `repository.memo` directly). Thin CRUD wrappers (`create_memo`/`get_memo`/`list_memos`/`count_memos`/`search_memos`/`update_memo`/`delete_memo`) over `repository.memo` — **all scoped to the connected `user`, no `is_admin`**. Centralizes the shared domain invariants: **title required + trim** on create/update (`TitleRequired`, `MemoError` base), and the **cross-entity rule that a memo's category must be one the owner registered** — `_require_known_category(user, category)` raises `UnknownCategory` unless the normalized category is `OTHERS` or exists via `repository.category.category_exists_db`. Each edge translates these (MCP → `Error: title is required.` / `Error: category 'X' is not registered. …`, web → HTTP 400). `summary` is trimmed here; `category` normalization stays in the repository. Also carries **semantic search**: `semantic_search(user, query, limit, category)` embeds the query → fetches candidates (`list_memos_db`, own memos only) → get-or-computes each summary embedding → ranks by cosine similarity, desc. **The only domain layer allowed to make network calls.** Tests monkeypatch `service.memo.embed_text`. |
| `service/category.py` | Category domain layer (mirrors `service.user`). Wraps `repository.category` and enforces the category invariants, returning dicts or raising domain exceptions (`CategoryError` base: `CategoryNameRequired`, `CategoryAlreadyExists`, `CategoryNotFound`, `CannotModifyOthers`): `list_categories(user)`, `create_category` (required + normalized + dup check), `rename_category` (`OTHERS` forbidden, source-exists, collision check, then cascade rename), `delete_category` (`OTHERS` forbidden, then reassign-to-`OTHERS` + drop). Also `rename_category_by_id` / `delete_category_by_id` for the web layer (resolve id → name via `get_category_db`, then delegate). Holds no authorization; categories are per-user (no admin cross-user). |
| `service/user.py` | User-CRUD domain rules **shared by both the MCP tools and the web UI**. Wraps `repository.user` and enforces domain invariants (`name` required + trimmed, `display_name`/`note` trim + partial-update `None`=unchanged, `admin` not deletable, `name` immutable). On `create_user` success it seeds the new user's default category via `repository.category.ensure_default_category_db` (**new users start with only `OTHERS`**). `delete_user` relies on the repository's cascade (memos/categories/embeddings removed). Returns structured `dict`/`None` or raises domain exceptions (`UserError` base: `NameRequired`, `UserAlreadyExists`, `UserNotFound`, `CannotDeleteAdmin`) — **never strings**. **Holds no authorization**. `switch_user` is a session op, so it stays in `tools/user.py`. |
| `server/mcp/auth.py` | **Identification only** of the connected user. Transport (http vs stdio) is a startup-time constant: `main()` calls `set_http_transport()` once, and `transport_is_http()` reads that flag (no per-call `get_http_request()` try/except to "detect" the transport). `current_user()`: stdio → the value recorded by `set_stdio_user()`; HTTP → if `?client_id=` is present, looks up the in-memory `_http_user_by_client` map (initializing it from `?user=` via `setdefault`, so a prior `switch_user` value is preserved and middleware double-reads are idempotent), else falls back to `?user=` (backward compatible). In HTTP mode `get_http_request()` is called directly to read the request; if it ever raised outside a request context the error surfaces rather than silently degrading to stdio. Helpers: `http_client_id()`, `transport_is_http()`, `switch_http_user(client_id, target)`. The map is in-memory (reset on restart; no TTL — fine for personal/local use). |
| `server/mcp/authz.py` | **Authorization** (shared by both domains). `resolve_caller()` runs `current_user()` → registration check (`is_registered_user`) and returns `(user, is_admin, error)`. If `error` is set, the tool returns it and stops. Error constants (`NO_USER_ERROR` / `ADMIN_ONLY_ERROR` / `not_registered_error`) live here too. |
| `server/mcp/tools/__init__.py` | Side-effect import that loads the submodules (`category`, `memo`, `user`) to register the tools. |
| `server/mcp/tools/memo.py` | The 7 memo `@mcp.tool`s. Authorize via `resolve_caller()` (uses only the connected `user`; `is_admin` is ignored — memos are never cross-user), then call `service.memo` (aliased `*_service`) — **no direct `repository.memo` calls**. Results are JSON strings. `create_memo`/`update_memo` catch `TitleRequired` → `Error: title is required.` and `UnknownCategory` → `Error: category 'X' is not registered. Create it first.`; `semantic_search_memos` turns `EmbeddingError` into a message. (The comma-split of `search_memos`' `query` into OR keywords stays here.) |
| `server/mcp/tools/category.py` | The 2 category `@mcp.tool`s (**create + list only**; no update/delete over MCP). Authorize via `resolve_caller()` and operate on the connected user's categories via `service.category`. `create_category` translates `CategoryNameRequired` / `CategoryAlreadyExists` to messages; `list_categories` returns a JSON array. |
| `server/mcp/tools/user.py` | The 5 user-management `@mcp.tool`s (admin-only, each tagged `tags={ADMIN_TOOL_TAG}` so the visibility policy in `admin_tools.py` applies) plus `switch_user` (not admin-only, untagged). `switch_user` is `async` and takes an injected `ctx: Context`; after rewriting the current user it calls `apply_session_visibility(ctx, target)` to enable/disable the admin tools **for that session** (admin → enable, otherwise → disable). Each management tool still enforces `is_admin` (returns `ADMIN_ONLY_ERROR` otherwise) as a backstop behind the visibility layer, then calls `service.user` and translates its domain exceptions to the legacy message strings (so `delete_user`'s `admin`-undeletable rule now lives in `service.user`, not here). `switch_user` only requires `resolve_caller()` to succeed and `target` to be registered, then rewrites stdio user or the HTTP `client_id → user` map; its success message appends the target's category list via `service.category.list_categories(target)` (per-user; not the repository directly). |
| `server/mcp/admin_tools.py` | **Visibility policy for the admin-tagged tools, using FastMCP's official Component Visibility API** (https://gofastmcp.com/servers/visibility) — *not* middleware. Constant `ADMIN_TOOL_TAG = "admin"` (the 5 `*_user` tools carry it via `@mcp.tool(tags={ADMIN_TOOL_TAG})`). `apply_server_default(mcp)` calls `mcp.disable(tags={ADMIN_TOOL_TAG})` at startup — the **server-level** default, so every fresh session starts with the admin tools hidden (and calling one by name returns FastMCP's native `Unknown tool` error). `apply_session_visibility(ctx, current_user)` is the **session-level** toggle the `switch_user` tool calls after a switch: `ctx.enable_components(tags={ADMIN_TOOL_TAG})` when the new user is `admin` (and auto-enable is on), else `ctx.disable_components(...)`. FastMCP emits `notifications/tools/list_changed` for these automatically, so the client refreshes its tool list — the key reason this is *not* done with a `tools/list`-rewriting middleware. `admin_tools_auto_enable()` reads `MEMO_ADMIN_TOOLS_AUTO_ENABLE` (env-first, `mcps/memo/.env` fallback via `load_dotenv(parents[4]/.env)`; default on; `0`/`false`/`no`/`off` = off) — the safety switch that suppresses the session enable so the tools stay disabled even for `admin`. This visibility layer sits in front of each tool's own `resolve_caller()`/`is_admin` check (defense in depth), not as a replacement. |
| `server/mcp/logging_middleware.py` | `AuditLogMiddleware` (registered via `mcp.add_middleware`). Overrides `Middleware.on_message` — the outermost hook, so it captures every method incl. `initialize`. INFO: one line per `tools/call` (`tool/user/client_id/session`). DEBUG: full session picture for all methods (`method/request_id/client_id/raw_user/resolved_user/session`). `session` is the raw `Mcp-Session-Id` header. Resolves the user via `auth.current_user()`. |
| `server/web/app.py` | **Standalone user-management web UI** app body (`memo-admin`). A Starlette app (`create_app()`) serving `static/admin.html` and a REST layer. User handlers call `service.user` (400 `NameRequired`, 409 `UserAlreadyExists`, 404 `UserNotFound`, 403 `CannotDeleteAdmin`, 201 on create). **Per-user memo CRUD** under `/api/users/{name}/memos` calls `service.memo` (aliased `*_service`; handlers do HTTP-body parsing then translate `TitleRequired` → 400 and `UnknownCategory` → 400): paginated `GET …/memos?page=&per_page=&category=`, `POST`/`PUT` (category must be registered for that user, empty → `OTHERS`), `DELETE`. **Per-user category CRUD** under `/api/users/{name}/categories` calls `service.category`: `GET` (list), `POST` (create; 400 `CategoryNameRequired` / 409 `CategoryAlreadyExists`), `PUT …/{category_id:int}` (rename; 403 `CannotModifyOthers` / 404 `CategoryNotFound` / 409, cascades to memos), `DELETE …/{category_id:int}` (403/404, reassigns memos to `OTHERS`). All memo/category handlers scope to the path `name` (a user can never touch another's data: cross-user → 404). `_int_param` parses paging params; `per_page` clamped to `MAX_PER_PAGE` (=100), default `DEFAULT_PER_PAGE` (=20). Holds no domain logic itself; unauthenticated — same local-only security model as `switch_user`. |
| `server/web/main.py` | The `memo-admin` entry point. `main()` calls `init_db()` then `uvicorn.run(create_app(), ...)`, binding `MEMO_ADMIN_HOST` (default `127.0.0.1`) / `MEMO_ADMIN_PORT` (default `8090`); warns when bound off-localhost. |
| `server/web/static/admin.html` / `admin.css` / `admin.js` | The admin UI (vanilla HTML/CSS/JS, no build step — same approach as `dynamic_prompt/static`). `admin.js` calls the `/api/users` REST endpoints; `name` is immutable so only `display_name`/`note` are editable, and the `admin` row has no delete button. Each user row has a **メモ** button that opens a paginated memo panel (前へ/次へ + page/total indicator, `PER_PAGE`=20). The list rows are **read-only**; create and edit happen on a separate editor view (`#memo-edit-card`) that replaces the list, giving a screen-transition feel without a router (a second `<section>` toggled via `hidden`). 新規メモ opens the editor in create mode; a row's 編集 button opens it in edit mode pre-filled with that memo; both submit the same `#memo-edit-form` (title `<input>` + category **`<select>`** populated from the user's categories + summary `<textarea>`) → `POST …/memos` (create) or `PUT …/memos/{id}` (edit). The memo list shows a category column (badge) and a カテゴリ絞り込み **`<select>`** above the table (all-categories + each) that reloads page 1 with `?category=` on change; the active filter is kept in `memoView.category`. A **カテゴリ管理** button opens a per-user category panel (`#category-card`): list + add form + per-row rename(保存)/delete via `…/categories` endpoints; the `OTHERS` row has no rename/delete controls. `memoCategories` caches the open user's category names (loaded by `loadCategories`) and `fillCategoryOptions` fills the selects. 削除 stays an inline confirm. After create the panel reloads page 1; after edit it returns to the original page; キャンセル/戻る re-shows the list without reloading. |
| `server/mcp/app.py` | The `mcp` FastMCP instance (with `AuditLogMiddleware` added right after creation, and `apply_server_default(mcp)` called **after** the tools side-effect import so the admin tags exist to disable), the side-effect import of `memo.server.mcp.tools`, the module-level `init_db()`, and the `/health` `@mcp.custom_route`. Split from `main.py` so tools can `from memo.server.mcp.app import mcp` without importing the entry point (one-way: app imports tools; main imports app). |
| `server/mcp/main.py` | The MCP server entry point (`main()`). Reads `--user`/`MEMO_USER` and `--debug`/`MEMO_LOG_DEBUG` via argparse; calls `_configure_logging(debug)` (basicConfig to **stderr**; root stays INFO, only `memo.*` goes DEBUG so library noise is excluded). Imports `app` to get the `mcp` instance; `TRANSPORT` switches stdio/http and is recorded once via `auth.set_http_transport()`; HTTP `HOST` defaults to `127.0.0.1` (local-only; see security note). |
| `tests/conftest.py` | Shared pytest setup. Redirects the DB path to a temp file, empties the tables (incl. `categories`) before each test, and re-seeds `admin` plus its `OTHERS` category. Connections come from `_connect_db` so FK enforcement is on. Repository/service tests that own memos/categories for `alice`/`bob` register those users in a module-local `autouse` fixture (depending on `clean_tables` for ordering) because the FK on `*.user → users(name)` now rejects inserts for unregistered owners. |
| `tests/test_memo_repository.py` | Unit tests for `repository.memo` (CRUD + search + user isolation + category column normalization/filtering on the `memos` row). No `is_admin`/orphan tests (cross-user access removed). Category CRUD/列挙 lives in `test_category_repository.py`. |
| `tests/test_category_repository.py` | Unit tests for `repository.category` (normalize rules; first-class CRUD: create/dup, list-dict/sorted/per-user, `category_exists_db`, `ensure_default_category_db`, `get_category_db` scoping, rename **cascade to memos** (now via trigger) + isolation, delete **reassigns memos to `OTHERS`** (via trigger)). Registers `alice`/`bob` in an `autouse` fixture (FK requires owners). |
| `tests/test_category_service.py` | Tests for category domain rules: `service.category` invariants (required/dup/`OTHERS`-protected/collision/not-found), `service.memo` create/update **rejecting unregistered categories** (`UnknownCategory`), and `service.user.create_user` seeding only `OTHERS` + `delete_user` cascade. |
| `tests/test_user_repository.py` | Unit tests for `repository.user` (ledger CRUD + registration check + `delete_user_db` **FK cascade** of memos/categories/embeddings, and that it leaves other users untouched). |
| `tests/test_embedding_repository.py` | Unit tests for `repository.embedding` (cache get / upsert / overwrite / delete). A `memo_id` fixture creates a real owner+memo first (FK `memo_id → memos(id)`). |
| `tests/test_migrations.py` | Unit tests for the migration framework (`runner` version management + idempotency) and `m001_foreign_keys` against a hand-built **legacy no-FK** temp DB: adds the FKs, drops orphans, preserves valid rows, and that user-delete cascade + category rename/delete triggers work post-migration. Uses its own temp sqlite connections (not conftest's DB). |
| `tests/test_semantic_search.py` | Unit tests for `service.memo.semantic_search`. Monkeypatches `service.memo.embed_text` to fixed vectors and verifies ranking, cache hits, recompute on summary change, and user isolation (no cross-user; no network / API key needed). |
| `tests/test_admin_server.py` | Tests for the `memo-admin` REST layer via Starlette's `TestClient`: user CRUD (+name-required/dup-409/missing-404/partial/admin-undeletable-403), HTML serving, per-user memo listing + CRUD (incl. cross-user-404, unregistered-category-400), per-user memo category set/filter, and **category endpoints** (`/api/users/{name}/categories`: list/new-user-OTHERS-only, create/required-400/dup-409/missing-user-404, rename cascade + `OTHERS`-403 + collision-409 + missing-404, delete reassign-to-`OTHERS` + `OTHERS`-403 + missing-404). Reuses `conftest.py`'s temp DB. |
| `tests/test_mcp_client.py` | Integration tests through an MCP client (tool registration incl. `create_category`/`list_categories`; authorization; user management; semantic search; category filter roundtrip; **`create_memo` rejecting unregistered categories**; **admin cannot see other users' memos**; monkeypatches `service.memo.embed_text`. Runs via `asyncio.run()`). Also covers `switch_user` (stdio, incl. the target-category list), the `current_user` HTTP `client_id` branches (monkeypatching `auth.get_http_request`), the audit log (`caplog`), and the **admin tool visibility** (`admin_tools.py`): fresh session lists only `NON_ADMIN_TOOLS` and `Unknown tool` on admin tool by name; after `switch_user("admin")` in-session lists `EXPECTED_TOOLS` and can call them, switching to non-admin re-hides; `MEMO_ADMIN_TOOLS_AUTO_ENABLE` falsy keeps them hidden. Helpers `_switch_then_list` / `_switch_then_call` keep switch+action in one session. |

`server/mcp/app.py` creates the `mcp` instance and adds `AuditLogMiddleware`, then imports `memo.server.mcp.tools` via side-effect import; `tools/__init__.py` loads the submodules (`tools.memo` / `tools.user`) to register all tools. `server/mcp/main.py` imports `app` and provides the entry point. `init_db()` is called at module level in `app.py`, so the DB is reliably initialized on every startup path.

## Layer dependency direction

`server/{mcp,web}` (transport edges) → `service/*` → `repository/*` → `infra/database`, one-way. **Transport edges go through `service/*` for all domain data access — they do not call `repository/*` directly.** (The one deliberate exception is the registration check: `authz` and `switch_user` call `repository.user.is_registered_user`, an identity/authorization concern the architecture intentionally keeps at the authz layer, not memo data CRUD.) The MCP edge adds an authorization step before service: `tools/* → authz → service/* → infra`. `authz` uses `auth` (identification) and `repository.user` (registration check). Within `repository/*`, `repository.memo → repository.category` (for `normalize_category`) is kept one-way (and cycle-free because `repository.category` imports only from `infra`). Cross-table cascades are no longer written in `repository/*` at all — they live in the DB (FK `ON DELETE CASCADE` + the category↔memo triggers defined in `infra.database`), so `repository.category`'s rename/delete and `repository.user`'s delete touch a single table each. `service.memo` and `service.user` also reach `repository.category` (category validation / `OTHERS` seeding) — still `service → repository`, one-way. `infra/database` depends on no domain layer; it imports `migrations.run_migrations` **lazily inside `init_db()`** (the only allowed `infra → migrations` reference) so `migrations.m001` can import `infra.database` (`_create_triggers`, `OTHERS_CATEGORY`) without an import cycle. To add a new domain, add `repository/<domain>.py`, optionally `service/<domain>.py` for domain rules, and `tools/<domain>.py`, then add the load to `tools/__init__.py`. `logging_middleware` sits inside the MCP edge (it reads `auth` only) and is orthogonal to these layers. `admin_tools` is also part of the MCP edge: it holds no domain logic, only the FastMCP visibility policy (`mcp.disable` at startup + `ctx.*_components` driven by `switch_user`).

The two transport edges share domain logic but not authorization: both `server/mcp/tools/{memo,user,category}.py` and `server/web/app.py` go through `service/{memo,user,category}.py` (the DRY home for the CRUD invariants — user `name`-required, memo `title`-required + trim + registered-category, category required/`OTHERS`-protected/cascade), but each enforces (or omits) authorization itself — the MCP tools require the connected user via `resolve_caller()` (and `admin` for the `*_user` tools; memo/category tools are own-data-only and ignore `is_admin`), while the web UI is the unauthenticated local-admin surface. `service/*` holds no authorization, only domain invariants.

Semantic search involves the network (OpenAI call) plus ranking, so it is not placed in the `repository` layer; the **`service` layer** carries it (alongside the plain memo CRUD wrappers): `tools/memo.py` → `service.memo` → (`infra.embedding`'s OpenAI call + `repository.embedding` / `repository.memo`). This keeps the invariant that `repository/*` is pure SQLite only, no network.

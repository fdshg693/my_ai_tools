# Folder Structure

## Module structure

Files are split by domain (memo / user / category) and by layer: `infra/` (shared gateways — SQLite, OpenAI), `repository/` (pure data access), `service/` (domain rules + orchestration), and `server/` (the transport edges — `server/mcp/` for the MCP server incl. `auth`/`authz`/`tools`, `server/web/` for the local admin web UI).

| Module | Responsibility |
|--------|---------------|
| `infra/database.py` | Shared infrastructure only. SQLite connection factory (`_connect_db`), `init_db()` (startup schema init + lightweight `user`-column migration + `users` / `memo_embeddings` table creation + `admin` seed), constant `ADMIN_USER = "admin"`. `DB_PATH` defaults to `parent.parent / "memo.db"` (= `src/memo/memo.db`), overridable via `MEMO_DB_PATH`. Holds no domain CRUD. |
| `repository/memo.py` | Pure data access for memos (`*_memo_db`). get/list/search/update/delete take a trailing `is_admin`; when True the user filter is dropped to cover all memos (including orphans). `list_memos_db` also takes `offset` for paging; `count_memos_db(user, is_admin)` returns the total for the same filter (used by the web memo-list pager). The category canonicalization rule lives in `repository/category.py`; `create_memo_db`/`update_memo_db` import `normalize_category()` from there and apply it on every write so stored values are canonical, and list/count/search take an optional `category` filter (built once via the private `_base_filter` helper shared by all three — `None` = all categories). Uses only `_connect_db` (+ `repository.category` for normalization); no authorization. |
| `repository/category.py` | Pure data access for categories, split out of `repository/memo.py` so category logic can grow independently. `normalize_category()` (trim + uppercase, empty/`None` → `OTHERS = OTHERS_CATEGORY`) — the canonical-form rule imported by `repository.memo` (one-way `memo → category`). `list_categories_db(user, is_admin)` returns the user's distinct categories present on their memos (name-sorted; `is_admin` drops the user filter). To avoid a `memo ↔ category` import cycle it does **not** use `memo._base_filter`; it builds the `user` clause inline and imports only from `infra` (`OTHERS_CATEGORY`, `_connect_db`). No authorization. |
| `repository/user.py` | Data access for the user ledger (`users`, `*_user_db`) plus the registration check (`is_registered_user`). `name` is the immutable identifier; `display_name`/`note` are editable. Deleting a user does not delete their memos. |
| `repository/embedding.py` | Pure data access for the embedding cache (`memo_embeddings`): `get_cached_embedding` / `upsert_embedding` (UPSERT) / `delete_embedding`. vector is a JSON array. No network, no authorization. |
| `infra/embedding.py` | OpenAI embedding API wrapper (leaf). The only place that imports `openai` and reads `OPENAI_API_KEY`. Provides `embed_text(text)`, constant `MODEL` (overridable via `MEMO_EMBEDDING_MODEL`), and exception `EmbeddingError`. The key is fetched lazily at call time. On import it loads `mcps/memo/.env` (fixed `parents[3]/.env`) via `python-dotenv` (existing env vars take precedence; .env does not override). |
| `service/memo.py` | Memo domain layer that **both transport edges go through** (MCP tools + web UI never call `repository.memo` directly). Thin CRUD wrappers (`create_memo`/`get_memo`/`list_memos`/`count_memos`/`search_memos`/`update_memo`/`delete_memo`) over `repository.memo`, plus the one shared domain invariant centralized here: **title required + trim** on create/update, raising `TitleRequired` (`MemoError` base) — each edge translates it (MCP → `Error: title is required.`, web → HTTP 400), mirroring `service.user`'s `NameRequired`. `summary` is trimmed here too; `category` is passed straight through (repository normalizes it). Also carries **semantic search**: `semantic_search(user, query, limit, is_admin, category)` embeds the query → fetches candidates (`list_memos_db`) → get-or-computes each summary embedding (`_embedding_for_memo`) → ranks by cosine similarity (`_cosine`, pure Python), desc. **The only domain layer allowed to make network calls** (repositories must not). Holds no authorization (`is_admin` is passed in). Tests monkeypatch `service.memo.embed_text`. |
| `service/category.py` | Category domain layer (thin). `list_categories(user, is_admin)` wraps `repository.category.list_categories_db` so the transport edge (`switch_user`) reaches categories via service rather than the repository. Holds no authorization; the home for future category domain rules (rename/merge etc.). |
| `service/user.py` | User-CRUD domain rules **shared by both the MCP tools and the web UI** (the DRY home for what used to be duplicated). Wraps `repository.user` and enforces domain invariants (`name` required + trimmed, `display_name`/`note` trim + partial-update `None`=unchanged semantics, `admin` not deletable, `name` immutable). Returns structured `dict`/`None` or raises domain exceptions (`UserError` base: `NameRequired`, `UserAlreadyExists`, `UserNotFound`, `CannotDeleteAdmin`) — **never strings**, so each caller maps them to its own surface (MCP → message strings; web → HTTP status). **Holds no authorization** (admin-only is enforced by the caller, not here). `switch_user` is a session op, not CRUD, so it stays in `tools/user.py`. |
| `server/mcp/auth.py` | **Identification only** of the connected user. Transport (http vs stdio) is a startup-time constant: `main()` calls `set_http_transport()` once, and `transport_is_http()` reads that flag (no per-call `get_http_request()` try/except to "detect" the transport). `current_user()`: stdio → the value recorded by `set_stdio_user()`; HTTP → if `?client_id=` is present, looks up the in-memory `_http_user_by_client` map (initializing it from `?user=` via `setdefault`, so a prior `switch_user` value is preserved and middleware double-reads are idempotent), else falls back to `?user=` (backward compatible). In HTTP mode `get_http_request()` is called directly to read the request; if it ever raised outside a request context the error surfaces rather than silently degrading to stdio. Helpers: `http_client_id()`, `transport_is_http()`, `switch_http_user(client_id, target)`. The map is in-memory (reset on restart; no TTL — fine for personal/local use). |
| `server/mcp/authz.py` | **Authorization** (shared by both domains). `resolve_caller()` runs `current_user()` → registration check (`is_registered_user`) and returns `(user, is_admin, error)`. If `error` is set, the tool returns it and stops. Error constants (`NO_USER_ERROR` / `ADMIN_ONLY_ERROR` / `not_registered_error`) live here too. |
| `server/mcp/tools/__init__.py` | Side-effect import that loads the submodules (`memo`, `user`) to register the tools. |
| `server/mcp/tools/memo.py` | The 7 memo `@mcp.tool`s. Authorize via `resolve_caller()`, then call `service.memo` (imported aliased as `*_service` to avoid colliding with the same-named tool functions) — **no direct `repository.memo` calls**. Results are returned as JSON strings. `create_memo`/`update_memo` catch `TitleRequired` → `Error: title is required.`; `semantic_search_memos` calls `service.memo.semantic_search` and turns `EmbeddingError` into a message. (The comma-split of `search_memos`' `query` into OR keywords stays here — input parsing — then calls `service.memo.search_memos`.) |
| `server/mcp/tools/user.py` | The 5 user-management `@mcp.tool`s (admin-only, each tagged `tags={ADMIN_TOOL_TAG}` so the visibility policy in `admin_tools.py` applies) plus `switch_user` (not admin-only, untagged). `switch_user` is `async` and takes an injected `ctx: Context`; after rewriting the current user it calls `apply_session_visibility(ctx, target)` to enable/disable the admin tools **for that session** (admin → enable, otherwise → disable). Each management tool still enforces `is_admin` (returns `ADMIN_ONLY_ERROR` otherwise) as a backstop behind the visibility layer, then calls `service.user` and translates its domain exceptions to the legacy message strings (so `delete_user`'s `admin`-undeletable rule now lives in `service.user`, not here). `switch_user` only requires `resolve_caller()` to succeed and `target` to be registered, then rewrites stdio user or the HTTP `client_id → user` map; its success message appends the target's category list via `service.category.list_categories` (not the repository directly). |
| `server/mcp/admin_tools.py` | **Visibility policy for the admin-tagged tools, using FastMCP's official Component Visibility API** (https://gofastmcp.com/servers/visibility) — *not* middleware. Constant `ADMIN_TOOL_TAG = "admin"` (the 5 `*_user` tools carry it via `@mcp.tool(tags={ADMIN_TOOL_TAG})`). `apply_server_default(mcp)` calls `mcp.disable(tags={ADMIN_TOOL_TAG})` at startup — the **server-level** default, so every fresh session starts with the admin tools hidden (and calling one by name returns FastMCP's native `Unknown tool` error). `apply_session_visibility(ctx, current_user)` is the **session-level** toggle the `switch_user` tool calls after a switch: `ctx.enable_components(tags={ADMIN_TOOL_TAG})` when the new user is `admin` (and auto-enable is on), else `ctx.disable_components(...)`. FastMCP emits `notifications/tools/list_changed` for these automatically, so the client refreshes its tool list — the key reason this is *not* done with a `tools/list`-rewriting middleware. `admin_tools_auto_enable()` reads `MEMO_ADMIN_TOOLS_AUTO_ENABLE` (env-first, `mcps/memo/.env` fallback via `load_dotenv(parents[4]/.env)`; default on; `0`/`false`/`no`/`off` = off) — the safety switch that suppresses the session enable so the tools stay disabled even for `admin`. This visibility layer sits in front of each tool's own `resolve_caller()`/`is_admin` check (defense in depth), not as a replacement. |
| `server/mcp/logging_middleware.py` | `AuditLogMiddleware` (registered via `mcp.add_middleware`). Overrides `Middleware.on_message` — the outermost hook, so it captures every method incl. `initialize`. INFO: one line per `tools/call` (`tool/user/client_id/session`). DEBUG: full session picture for all methods (`method/request_id/client_id/raw_user/resolved_user/session`). `session` is the raw `Mcp-Session-Id` header. Resolves the user via `auth.current_user()`. |
| `server/web/app.py` | **Standalone user-management web UI** app body (`memo-admin`). A Starlette app (built by `create_app()`) serving an admin page (`static/admin.html`) and a `/api/users` REST layer. Each handler calls `service.user` and maps its return value / domain exceptions to HTTP status (400 `NameRequired`, 409 `UserAlreadyExists`, 404 `UserNotFound`, 403 `CannotDeleteAdmin`, 201 on create). Also exposes a **per-user memo CRUD** surface under `/api/users/{name}/memos` (calls `service.memo`, aliased `*_service` — **not** `repository.memo` directly — so the title-required + trim invariant is shared with the MCP tools; the handlers only do HTTP-body parsing then translate `TitleRequired` → 400): `GET …/memos?page=&per_page=&category=` (paginated list; optional `category` filter), `POST …/memos` (create; `title` required → 400, owner fixed to the path `name`, optional `category` normalized, 201), `PUT …/memos/{memo_id:int}` (partial update of `title`/`summary`/`category`; empty `title` → 400; omitted `category` unchanged, empty string → `OTHERS`), `DELETE …/memos/{memo_id:int}`. All memo handlers scope to that one user via `is_admin=False` (so a user can never touch another's memo: cross-user `PUT`/`DELETE` 404 by the same not-found-=-not-leaked rule); create/list 404 if the user is absent, update/delete 404 if the memo isn't found under that user. `_int_param` parses paging query params with fallback to defaults; `per_page` is clamped to `MAX_PER_PAGE` (=100), default `DEFAULT_PER_PAGE` (=20). A transport edge that holds no domain logic itself; the `admin`-undeletable guard now lives in `service.user` (shared with the MCP tool). Unauthenticated — same local-only security model as `switch_user`. |
| `server/web/main.py` | The `memo-admin` entry point. `main()` calls `init_db()` then `uvicorn.run(create_app(), ...)`, binding `MEMO_ADMIN_HOST` (default `127.0.0.1`) / `MEMO_ADMIN_PORT` (default `8090`); warns when bound off-localhost. |
| `server/web/static/admin.html` / `admin.css` / `admin.js` | The admin UI (vanilla HTML/CSS/JS, no build step — same approach as `dynamic_prompt/static`). `admin.js` calls the `/api/users` REST endpoints; `name` is immutable so only `display_name`/`note` are editable, and the `admin` row has no delete button. Each user row has a **メモ** button that opens a paginated memo panel (前へ/次へ + page/total indicator, `PER_PAGE`=20). The list rows are **read-only**; create and edit happen on a separate editor view (`#memo-edit-card`) that replaces the list, giving a screen-transition feel without a router (a second `<section>` toggled via `hidden`). 新規メモ opens the editor in create mode; a row's 編集 button opens it in edit mode pre-filled with that memo; both submit the same `#memo-edit-form` (title `<input>` + category `<input>` + summary `<textarea>`) → `POST …/memos` (create) or `PUT …/memos/{id}` (edit). The memo list shows a category column (badge) and a カテゴリ絞り込み input above the table (絞り込み/クリア buttons + Enter) that reloads page 1 with `?category=`; the active filter is kept in `memoView.category` so paging and post-edit reloads preserve it (opening a user via メモ resets it). 削除 stays an inline confirm + `DELETE …/memos/{id}` on the list row. After create the panel reloads page 1 (newest-first); after edit it returns to the original page; キャンセル/戻る re-shows the list without reloading. |
| `server/mcp/app.py` | The `mcp` FastMCP instance (with `AuditLogMiddleware` added right after creation, and `apply_server_default(mcp)` called **after** the tools side-effect import so the admin tags exist to disable), the side-effect import of `memo.server.mcp.tools`, the module-level `init_db()`, and the `/health` `@mcp.custom_route`. Split from `main.py` so tools can `from memo.server.mcp.app import mcp` without importing the entry point (one-way: app imports tools; main imports app). |
| `server/mcp/main.py` | The MCP server entry point (`main()`). Reads `--user`/`MEMO_USER` and `--debug`/`MEMO_LOG_DEBUG` via argparse; calls `_configure_logging(debug)` (basicConfig to **stderr**; root stays INFO, only `memo.*` goes DEBUG so library noise is excluded). Imports `app` to get the `mcp` instance; `TRANSPORT` switches stdio/http and is recorded once via `auth.set_http_transport()`; HTTP `HOST` defaults to `127.0.0.1` (local-only; see security note). |
| `tests/conftest.py` | Shared pytest setup. Redirects the DB path to a temp file, empties the tables before each test, and re-seeds `admin`. |
| `tests/test_memo_repository.py` | Unit tests for `repository.memo` (CRUD + search + user isolation + admin privilege + category: list/count/search filtering, update set/keep/reset). The category helpers it also exercises — `normalize_category` rules / default `OTHERS`, `list_categories_db` distinct/sorted/per-user/admin/empty — now live in `repository.category` and are imported from there (one test file still covers both, since the data is the same `memos` table). |
| `tests/test_user_repository.py` | Unit tests for `repository.user` (ledger CRUD + registration check). |
| `tests/test_embedding_repository.py` | Unit tests for `repository.embedding` (cache get / upsert / overwrite / delete). |
| `tests/test_semantic_search.py` | Unit tests for `service.memo.semantic_search`. Monkeypatches `service.memo.embed_text` to fixed vectors and verifies ranking, cache hits, recompute on summary change, user isolation, and admin cross-user (no network / API key needed). |
| `tests/test_admin_server.py` | Tests for the `memo-admin` REST layer via Starlette's `TestClient` (httpx): user list/create/get/update/delete, name-required + duplicate (409) + missing (404) cases, partial update (only supplied fields), the `admin`-undeletable guard (403), HTML serving, the per-user memo listing (`/api/users/{name}/memos`: empty, missing-user 404, user-scoping, pagination, `per_page` cap, invalid-param fallback), and the per-user memo CRUD (create incl. trim/summary-default/title-required-400/missing-user-404, update incl. partial/empty-title-400/cross-user-404/missing-404, delete incl. cross-user-404/missing-404), plus category (create with/without category, update category, list `?category=` filter). Reuses `conftest.py`'s temp DB. |
| `tests/test_mcp_client.py` | Integration tests through an MCP client (tool registration + authorization + admin cross-user + user management + semantic search + category filter roundtrip via `create_memo`/`list_memos`/`search_memos`; monkeypatches `service.memo.embed_text`. Runs via `asyncio.run()` without depending on pytest-asyncio). Also covers `switch_user` (stdio path, incl. the target-category list in its success message), the `current_user` HTTP `client_id` branches (monkeypatching `auth.get_http_request` — i.e. `memo.server.mcp.auth` — with a dummy request), and the audit log (`caplog` for the INFO tool line and the DEBUG `initialize` line; the logger name is `memo.server.mcp.logging_middleware`), and the **admin tool visibility** (`admin_tools.py`): a fresh session (even one started as `admin`) lists only `NON_ADMIN_TOOLS` and gets `Unknown tool` when calling an admin tool by name (server-level default); after `switch_user("admin")` **in the same session** the connection lists `EXPECTED_TOOLS` and can call them, while `switch_user` to a non-admin re-hides them; setting `MEMO_ADMIN_TOOLS_AUTO_ENABLE` falsy keeps them hidden/uncallable even after switching to `admin`. Helpers `_switch_then_list` / `_switch_then_call` keep the switch and the action in one `Client` session (visibility is per-session). |

`server/mcp/app.py` creates the `mcp` instance and adds `AuditLogMiddleware`, then imports `memo.server.mcp.tools` via side-effect import; `tools/__init__.py` loads the submodules (`tools.memo` / `tools.user`) to register all tools. `server/mcp/main.py` imports `app` and provides the entry point. `init_db()` is called at module level in `app.py`, so the DB is reliably initialized on every startup path.

## Layer dependency direction

`server/{mcp,web}` (transport edges) → `service/*` → `repository/*` → `infra/database`, one-way. **Transport edges go through `service/*` for all domain data access — they do not call `repository/*` directly.** (The one deliberate exception is the registration check: `authz` and `switch_user` call `repository.user.is_registered_user`, an identity/authorization concern the architecture intentionally keeps at the authz layer, not memo data CRUD.) The MCP edge adds an authorization step before service: `tools/* → authz → service/* → infra`. `authz` uses `auth` (identification) and `repository.user` (registration check). Within `repository/*`, `repository.memo → repository.category` (for `normalize_category`) is the only cross-module edge, kept one-way (and cycle-free because `repository.category` imports only from `infra`). `infra/database` depends on no domain layer. To add a new domain, add `repository/<domain>.py`, optionally `service/<domain>.py` for domain rules, and `tools/<domain>.py`, then add the load to `tools/__init__.py`. `logging_middleware` sits inside the MCP edge (it reads `auth` only) and is orthogonal to these layers. `admin_tools` is also part of the MCP edge: it holds no domain logic, only the FastMCP visibility policy (`mcp.disable` at startup + `ctx.*_components` driven by `switch_user`).

The two transport edges share domain logic but not authorization: both `server/mcp/tools/{memo,user}.py` and `server/web/app.py` go through `service/{memo,user}.py` (the DRY home for the CRUD invariants — user `name`-required, memo `title`-required + trim), but each enforces (or omits) authorization itself — the MCP tools require the connected user via `resolve_caller()` (and `admin` for the `*_user` tools), while the web UI is the unauthenticated local-admin surface (matching `switch_user`'s security model; its own process has no MCP connection to identify). `service/*` holds no authorization, only domain invariants.

Semantic search involves the network (OpenAI call) plus ranking, so it is not placed in the `repository` layer; the **`service` layer** carries it (alongside the plain memo CRUD wrappers): `tools/memo.py` → `service.memo` → (`infra.embedding`'s OpenAI call + `repository.embedding` / `repository.memo`). This keeps the invariant that `repository/*` is pure SQLite only, no network.

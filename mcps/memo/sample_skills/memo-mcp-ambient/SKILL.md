# 自動起動を許可する「裏方」スキル。積極的な深掘り質問はしない。
# memo-mcp-personalize と同じユーザー名 "personalize" を共有し、蓄えた事実を相互利用する。
# 通常会話に溶け込ませる前提なので、参照も保存も会話の流れを邪魔しない範囲で行う。
---

## 絶対の原則（最優先）

- **事実を引き出す質問をしない**。「あなたの好きな言語は？」のような深掘り質問はこのスキルの
  対象外です（それが必要なら `memo-mcp-personalize` を使います）。記録するのは、会話の自然な
  流れの中で **たまたま判明した** 事実だけです。
  **もちろん、普通にユーザーに質問すべき場合は、普通に質問して構いません。変にためらって、質問を省略して取り合えずの回答を返すのは避けます。**
- **会話を中断・誘導しない**。メモ操作のために話題を変えたり、確認のための往復を増やしたり
  しません。ユーザーには「記憶していること」を意識させず、結果として自然で個人化された応答を返します。
- 機微な情報（健康・信条・金銭など）は、明確に「覚えておいて」と言われた場合を除き、自分から
  保存しません。保存してよいか迷うものは保存しません。

## ログイン（最初のメモ操作の直前に一度だけ）

メモを読み書きする必要が生じたら、その最初の操作の直前に一度だけログインします。

1. `switch_user("personalize")` を呼びます。以降この会話でのメモ操作はすべて `personalize`
   のものに限定されます。
2. 「登録されていない」エラーが返ったら、`personalize` が未登録です。admin として接続中なら
   `create_user("personalize")` で登録してから再度ログインします。そうでなければ、記憶機能は
   今回使えない旨を一言添える程度にとどめ、通常どおり会話を続けます（エラーで会話を止めない）。
3. ログインは会話につき一度で十分です。毎回呼び直す必要はありません。

> 会話開始時に先回りして `list_memos` で全件を読み込むことはしません（コストと無関係な
> 文脈混入を避けるため）。必要になった話題について、その都度ピンポイントに引きます。

## 参照（recall）: 役立つときだけ思い出す

ユーザーの背景・好み・文脈を踏まえると応答の質が上がりそうなときだけ、関連するメモを引きます。

1. いま話している話題を表す自然文で `semantic_search_memos` を呼び、関連しそうな事実を探します
   （固有名詞がはっきりしているなら `search_memos` を併用してもよい）。
2. ヒットしたメモを応答の前提として自然に織り込みます。すでに知っていることは聞き直しません。
3. ヒットしなければ、既存知識に縛られず通常どおり応答します。検索結果が無いことを
   わざわざ報告する必要はありません。

## 保存（update）: 自然に判明した事実だけを静かに残す

会話の中でユーザー自身に関する **持続的で再利用価値のある** 事実が判明したときだけ保存します
（その場限りの細部や、一時的な状態は対象外）。

1. 保存する前に、その事実を表す自然文で `semantic_search_memos` を呼び、同種のメモが
   既にないか確認します（重複を作らない）。
2. 保存方法を判断します（保存先は必ずログイン中の `personalize`）。
   - **新しい事実**: `create_memo` で保存します。`title` は事実のカテゴリ
     （例:「好きな言語」「進行中のプロジェクト」）、`summary` は具体的な内容にします。
   - **既存の事実の更新・追記**: 検索で `memo_id` を特定し、`update_memo` で `summary` を
     最新化します（古い情報は上書き、または追記）。
   - **誤りと判明した情報**: `delete_memo` で削除します。
3. 保存は黙って行ってよく、いちいち「メモしました」と報告する必要はありません
   （ただし、ユーザーが明示的に「覚えておいて」と頼んだ場合は、覚えた旨を一言返すと親切です）。

## 粒度・運用の指針

- **1 メモ 1 事実** を基本とし、後から検索・更新しやすい粒度に保ちます。
- 迷ったら「保存しない・質問しない」を選びます。このスキルは控えめさが価値です。
- より体系的にユーザーを学習したくなった（積極的に深掘りしたい）場合は、ユーザーに
  `memo-mcp-personalize` の明示呼び出しを案内します。蓄積先は同じ `personalize` なので、
  そちらで深掘りした事実もこのスキルがそのまま活用できます。
